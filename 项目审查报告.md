## 🔴 严重问题（Critical / High）

### 1. 配置文件硬编码敏感信息

**文件**: `vending-server/src/main/resources/application.yml`

**问题描述**:
所有敏感信息（JWT密钥、数据库密码、MinIO密钥）都硬编码在 `application.yml` 中，存在严重安全风险：

```yaml
jwt:
  secret: vending-system-secret-key-2026-must-be-at-least-256-bits
  # ↑ 密钥写在配置文件里，一旦提交到Git就会泄露

spring:
  datasource:
    username: vending
    password: vending1234    # ↑ 数据库密码明文
    url: jdbc:mysql://localhost:3306/vending_db?...

minio:
  access-key: minioadmin
  secret-key: minioadmin1234  # ↑ MinIO密钥明文
```

**风险影响**:
- 配置文件如果被提交到Git，所有密钥会泄露
- 不同环境（开发/测试/生产）无法灵活切换配置
- 不符合安全最佳实践

**修复建议**:
```yaml
# 改为使用环境变量，本地开发可使用默认值
jwt:
  secret: ${JWT_SECRET:default-dev-secret-key-must-be-long}
  expiration: ${JWT_EXPIRATION:7200000}

spring:
  datasource:
    url: ${DB_URL:jdbc:mysql://localhost:3306/vending_db?useUnicode=true&characterEncoding=utf-8}
    username: ${DB_USERNAME:vending}
    password: ${DB_PASSWORD:}
    # 生产环境不设置默认值，强制要求配置

minio:
  endpoint: ${MINIO_ENDPOINT:http://localhost:9000}
  access-key: ${MINIO_ACCESS_KEY:minioadmin}
  secret-key: ${MINIO_SECRET_KEY:}
```

**优先级**: 🔴 P0（立即处理）

### 3. ProductController 缺少权限校验和输入验证

**文件**: `vending-server/src/main/java/com/vending/module/product/controller/ProductController.java`

**问题描述**:

#### 3.1 缺少输入验证
```java
@PostMapping
public Result<Void> save(@RequestBody Product product) {  // ← 无 @Valid 注解
    productService.save(product);
    redisCacheUtil.deleteByPrefix(RedisCacheUtil.KEY_PRODUCT_LIST);
    return Result.success();
}

@PutMapping
public Result<Void> update(@RequestBody Product product) { // ← 无 @Valid 注解
    productService.updateById(product);
    redisCacheUtil.deleteByPrefix(RedisCacheUtil.KEY_PRODUCT_LIST);
    return Result.success();
}
```

#### 3.2 缺少管理员权限校验
```java
@DeleteMapping("/{id}")
public Result<Void> delete(@PathVariable Long id) {  // ← 任何登录用户都能删除商品
    productService.removeById(id);
    redisCacheUtil.deleteByPrefix(RedisCacheUtil.KEY_PRODUCT_LIST);
    return Result.success();
}
```

#### 3.3 返回敏感字段
```java
@GetMapping("/{id}")
public Result<Product> getById(@PathVariable Long id) {
    Product product = productService.getById(id);
    return Result.success(product);  // ← 返回完整实体，可能暴露内部字段（如逻辑删除标记deleted）
}
```

**风险影响**:
- 恶意用户可提交任意字段（如伪造ID、状态等）
- 普通用户可以增删改商品信息
- 可能暴露数据库内部字段

**修复建议**:
```java
// 1. 使用专门的请求DTO，并添加验证注解
@PostMapping
public Result<Void> save(@RequestAttribute("role") Integer role, 
                       @Valid @RequestBody ProductCreateRequest request) {
    // 检查管理员权限
    if (role == null || role == 0) {  // 假设0是普通用户
        throw new BusinessException(ResultCode.FORBIDDEN, "无权限执行此操作");
    }
    productService.save(request);
    redisCacheUtil.deleteByPrefix(RedisCacheUtil.KEY_PRODUCT_LIST);
    return Result.success();
}

// 2. 返回视图对象，不暴露实体内部字段
@GetMapping("/{id}")
public Result<ProductVO> getById(@PathVariable Long id) {
    ProductVO product = productService.getProductVOById(id);
    return Result.success(product);
}
```

**优先级**: 🔴 P0（安全漏洞）

### 4. JWT过滤器吞掉所有异常

**文件**: `vending-server/src/main/java/com/vending/security/JwtAuthenticationFilter.java`

**问题描述**:
```java
try {
    // ... token解析和校验逻辑
    Long userId = jwtUtil.getUserId(token);
    String username = jwtUtil.parseToken(token).getSubject();
    Integer role = jwtUtil.getRole(token);
    // ...
} catch (Exception e) {
    // Token 无效，不设置认证信息，由 Spring Security 处理未授权
    // ↑ 问题：所有异常（包括代码bug）都被静默吞掉
}
```

**影响**:
- 任何异常（包括空指针、类型转换错误等bug）都被 `catch (Exception e)` 静默吞掉
- 调试困难，无法区分是JWT无效还是代码存在bug
- 可能隐藏真正的安全问题

**修复建议**:
```java
try {
    // ... token解析和校验逻辑
} catch (JwtException e) {
    // JWT相关异常：token无效或过期，这是预期行为
    log.debug("Invalid JWT token: {}", e.getMessage());
} catch (Exception e) {
    // 其他异常：可能是代码bug，需要记录
    log.error("Unexpected error during JWT authentication", e);
    // 可以选择返回500错误，而不是静默失败
}
```

**优先级**: 🟡 P1（调试和维护）

### 8. Redis缓存清除效率问题

**文件**: `vending-server/src/main/java/com/vending/common/cache/RedisCacheUtil.java`

**问题描述**:
```java
public void deleteByPrefix(String prefix) {
    ScanOptions options = ScanOptions.scanOptions()
            .match(prefix + "*")
            .count(100)  // ← count(100) 对于大量key可能不够高效
            .build();
    
    try (Cursor<String> cursor = redisTemplate.scan(options)) {
        while (cursor.hasNext()) {
            keys.add(cursor.next());
        }
    }
    
    if (!keys.isEmpty()) {
        redisTemplate.delete(keys);  // ← 一次性删除大量key可能导致Redis阻塞
    }
}
```

**影响**: 当缓存key数量很多时，一次性删除可能导致Redis短暂阻塞。

**修复建议**:
```java
public void deleteByPrefix(String prefix) {
    ScanOptions options = ScanOptions.scanOptions()
            .match(prefix + "*")
            .count(500)  // 增加批量大小
            .build();
    
    List<String> keys = new ArrayList<>();
    try (Cursor<String> cursor = redisTemplate.scan(options)) {
        while (cursor.hasNext()) {
            keys.add(cursor.next());
            // 分批删除，避免一次性删除过多key
            if (keys.size() >= 100) {
                redisTemplate.delete(keys);
                keys.clear();
            }
        }
    }
    
    if (!keys.isEmpty()) {
        redisTemplate.delete(keys);
    }
}
```

**优先级**: 🟡 P1

### 9. 缺少全局CORS配置

**问题描述**: 经过代码检查，没有发现显式的 Spring Security CORS 配置类。如果前端和后端不在同一域名下，会遇到跨域问题。

**修复建议**:
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration configuration = new CorsConfiguration();
        configuration.setAllowedOrigins(Arrays.asList("http://localhost:5173", "http://localhost:5174"));
        configuration.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "DELETE", "OPTIONS"));
        configuration.setAllowedHeaders(Arrays.asList("*"));
        configuration.setAllowCredentials(true);
        
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/**", configuration);
        return source;
    }
}
```

### 14. 前端错误提示可以更细致

**问题描述**: 当前错误提示使用 `ElMessage.error(res.message || '请求失败')`，但不同错误类型（400、403、500等）没有区分处理。

**修复建议**:
```javascript
request.interceptors.response.use(
  response => {
    const res = response.data
    if (res.code === 200) {
      return res
    } else {
      // 根据错误码显示不同提示
      const errorMessages = {
        400: '请求参数错误',
        401: '未授权，请重新登录',
        403: '拒绝访问',
        404: '请求资源不存在',
        500: '服务器内部错误'
      }
      const message = errorMessages[res.code] || res.message || '请求失败'
      ElMessage.error(message)
      
      if (res.code === 401) {
        const userStore = useUserStore()
        userStore.logout()
        router.push('/login')
      }
      return Promise.reject(new Error(res.message))
    }
  },
  error => {
    // 网络错误处理
    if (error.code === 'ECONNABORTED') {
      ElMessage.error('请求超时，请稍后重试')
    } else if (!error.response) {
      ElMessage.error('网络连接失败，请检查网络')
    } else {
      ElMessage.error(error.message || '网络错误')
    }
    return Promise.reject(error)
  }
)
```

**优先级**: 🟢 P2
