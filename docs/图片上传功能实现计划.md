## 核心功能说明

### 1. 后端实现

#### MinioService.java

主要功能：
- 自动创建 Bucket（如不存在）
- 自动设置 Bucket 为 public-read 权限
- 上传文件（支持 JPG/PNG/GIF 格式）
- 删除文件
- 获取预签名 URL（可选）

**文件限制**：
- 允许类型：`image/jpeg`, `image/jpg`, `image/png`, `image/gif`
- 最大大小：10MB（已在 application.yml 配置）
- 文件命名：使用 UUID 避免重名冲突

#### FileUploadController.java

API 接口：
- `POST /api/upload` - 上传文件
  - 参数：
    - `file` - 上传的文件
    - `prefix` - 文件前缀（可选，默认 `products`）
  - 返回：图片访问 URL

### 2. 前端实现

#### ProductManage.vue

功能：
- 支持图片预览
- 拖拽或点击上传
- 上传前格式和大小校验
- 删除已上传图片
- 编辑时显示已有图片

## MinIO 存储结构

```
vending-images/              # Bucket（默认名称）
└── products/                # 商品图片文件夹
    ├── uuid1.jpg
    ├── uuid2.png
    └── ...
```

## 图片 URL 格式

```
http://localhost:9000/vending-images/products/uuid-filename.jpg
```

## 配置说明

### application.yml 配置

```yaml
minio:
  endpoint: ${MINIO_ENDPOINT:http://localhost:9000}
  access-key: ${MINIO_ACCESS_KEY:minioadmin}
  secret-key: ${MINIO_SECRET_KEY:minioadmin1234}
  bucket-name: ${MINIO_BUCKET:vending-images}

spring:
  servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 10MB
```

### 环境变量

| 环境变量 | 说明 | 默认值 |
|---------|------|--------|
| `MINIO_ENDPOINT` | MinIO 服务地址 | `http://localhost:9000` |
| `MINIO_ACCESS_KEY` | MinIO 访问密钥 | `minioadmin` |
| `MINIO_SECRET_KEY` | MinIO 密钥 | `minioadmin1234` |
| `MINIO_BUCKET` | 存储桶名称 | `vending-images` |

## 使用指南

### 管理后台上传商品图片

1. 登录管理后台
2. 进入「商品管理」
3. 点击「新增商品」或「编辑」
4. 在「商品图片」区域点击上传
5. 选择图片文件（支持 JPG/PNG/GIF，最大 10MB）
6. 图片上传成功后自动保存 URL
7. 点击「保存」完成

### API 调用示例

```javascript
// 前端调用上传接口
const formData = new FormData();
formData.append('file', file);
formData.append('prefix', 'products');

const response = await fetch('/api/upload', {
  method: 'POST',
  body: formData
});

const result = await response.json();
const imageUrl = result.data; // 获取上传后的图片 URL
```

## 部署与初始化

### 1. 启动 MinIO

使用 Docker 启动（推荐）：
```bash
docker-compose up -d minio
```

### 2. 访问 MinIO 控制台

- 地址：http://localhost:9001
- 账号：minioadmin
- 密码：minioadmin1234

### 3. Bucket 自动创建

系统启动时会自动检查并创建 `vending-images` Bucket，并设置为 public-read 权限，无需手动操作。

## 注意事项

⚠️ **安全提示**：
1. 生产环境请修改 MinIO 默认密码
2. 图片访问权限设置为 public-read 是为了简化开发，生产环境可考虑使用预签名 URL
3. 建议在 CDN 前部署 MinIO 以加速访问

🛠️ **故障排查**：
- 图片上传失败：检查 MinIO 是否正常运行
- 图片无法访问：检查 Bucket 权限设置
- 跨域问题：确认 CorsConfig 已正确配置
