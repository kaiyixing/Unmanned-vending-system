## 三、实现步骤

### 步骤1：创建MinioService
- 封装上传文件方法
- 封装删除文件方法
- 封装获取文件访问URL方法
- 自动创建bucket逻辑

### 步骤2：创建FileUploadController
- 提供 `/api/upload` 接口
- 校验文件类型（仅图片）
- 调用MinioService上传
- 返回图片URL

### 步骤3：修改ProductManage.vue
- 在表单中添加el-upload组件
- 支持图片预览
- 上传成功后回填imageUrl
- 编辑时显示已有图片

### 步骤4：（可选）更新init_data.sql
- 给现有商品添加占位图片URL（使用在线占位图服务）

## 四、技术细节

### MinIO存储结构
```
vending-images/          ← bucket
└── products/            ← 文件夹
    ├── {uuid}.jpg       ← 商品图片（文件名用UUID避免重名）
    └── ...
```

### 图片URL格式
```
http://localhost:9000/vending-images/products/{uuid}.jpg
```

### 文件限制
- 允许类型：jpg, jpeg, png, gif
- 最大大小：10MB（已在application.yml配置）

## 六、风险与注意事项

### 风险1：MinIO未启动或bucket不存在
- 解决：代码中添加bucket自动创建逻辑

### 风险2：跨域问题
- 解决：已在项目中配置CORS

### 风险3：图片访问权限
- 解决：设置bucket为public-read，或使用预签名URL（先选简单方案：public-read）
